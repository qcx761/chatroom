| 模块            | 职责                                 |
| ------------- | ---------------------------------- |
| `server/`     | 最上层服务器调度模块，可保留现有接口用于整合其他模块         |
| `reactor/`    | 主从 Reactor（Epoll事件循环）逻辑，注册FD、处理事件等 |
| `handler/`    | 聊天、登录等消息处理器，供 `sub_reactor` 回调触发   |
| `threadpool/` | 处理登录、发消息、查数据库的逻辑，避免阻塞 I/O          |
| `model/`      | 用户、好友、群组的数据结构和抽象                   |
| `db/`         | Redis/MySQL 封装访问接口                 |
| `common/`     | 工具库，日志、字符串处理等                      |
